# RIC Industrial Brutalist UI

## Role

This is a visual **modifier**. Apply it only when the brief and primary executor support the direction.

## Direction

- Use strong grid logic, explicit information hierarchy, technical typography, exposed structure, and disciplined contrast.
- Preserve usability, accessibility, semantic states, and efficient scanning.
- Adapt density and responsiveness to the actual workflow instead of forcing one breakpoint rule.
- Use motion sparingly for state, data change, navigation, or system feedback.

## Boundaries

Do not turn all controls into decoration, sacrifice readability for attitude, introduce unrelated component libraries, or redirect the task. Independent visual review decides whether the direction fits.

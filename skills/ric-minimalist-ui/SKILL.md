# RIC Minimalist UI

## Role

This is a visual **modifier**, not a primary executor. Minimalism means removing noise while preserving complete behavior, states, accessibility, and business meaning.

## Direction

- Use clear hierarchy, careful typography, deliberate spacing, restrained color, and high-quality state design.
- Keep controls discoverable and information density appropriate to the workflow.
- Use real, existing, or generated assets only when they improve the approved direction.
- Preserve loading, empty, error, permission, selected, focus, and responsive states.

## Boundaries

Do not use placeholder image services, erase necessary actions, force excessive whitespace, or redirect ownership. Independent visual review decides whether restraint remains usable and distinctive.

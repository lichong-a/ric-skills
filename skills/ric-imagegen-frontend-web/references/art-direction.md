# Web Art Direction

Use this reference when creating or selecting a web visual direction.

## Direction Inputs

- Product, audience, offer, and primary action.
- Brand assets and constraints.
- Page type and content volume.
- Desired expression, trust level, and conversion intent.
- Target viewport, responsive expectations, and implementation stack.

## Direction Dimensions

Meaningful directions vary several of:

- Hierarchy and content order.
- Product framing.
- Hero architecture.
- Typography character.
- Section rhythm and density.
- Image language and material treatment.
- Interaction or motion implication.

Palette-only variation is not a separate direction.

## Product Specificity

A useful direction should show:

- The actual product, service, subject, or experience.
- A believable primary action.
- Content and proof appropriate to the audience.
- A visual system that could guide later sections.

Reject generic dashboards, invented brands, fake testimonials, fake metrics, or arbitrary UI panels used as decoration.

## Anti-Template Questions

- Does the direction communicate the brief before the logo is read?
- Is the focal point specific and inspectable?
- Is the composition meaningfully different from a common template?
- Do decorative effects express the brand or merely fill space?
- Can the direction be implemented responsively?
- Does it remain readable without presentation effects?

## Text Strategy

Critical copy should be supplied separately and added in code or design tooling when exact wording matters. Generated text may be used for rough concept composition, but it must not be treated as final truth.

# Web Generation Plan

Use this reference to plan prompts, assets, continuity, and targeted regeneration.

## Asset Manifest

For every planned generation, record:

- Purpose and placement.
- Direction and subject.
- Target dimensions and aspect ratio.
- Focal point and safe crop.
- Required continuity with other assets.
- Text policy.
- Destination and version.

## Prompt Structure

Describe:

1. Subject and product context.
2. Composition and hierarchy.
3. Visual language, material, lighting, and palette.
4. Target crop, aspect ratio, and negative space.
5. Required product-specific detail.
6. Exclusions such as watermark, accidental text, fake trademarks, or distorted UI.

Avoid long lists of fashionable effects without a clear direction.

## Quantity Decision

Generate only what the decision or implementation needs:

- Direction selection: usually three distinct boards or hero concepts.
- High-risk section: one or more targeted variants.
- Full implementation support: references only for sections whose visual language cannot be derived from the selected system.
- Asset replacement: only the failed or missing asset.

## Continuity

Keep a selected set coherent through:

- Shared visual concept.
- Compatible palette and lighting.
- Consistent product and subject treatment.
- Deliberate variation in composition and scale.

Do not force every asset into the same frame.

## Regeneration

Inspect each output and state the defect before regenerating. Change the prompt to address that defect. Do not regenerate the whole set because one asset failed.
